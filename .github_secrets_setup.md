## üîí **Security Best Practices**

1. **Never commit tokens to git**
   - Tokens are already in `.gitignore`
   - Use environment variables
   - Use GitHub Secrets for CI/CD

2. **Rotate tokens regularly**
   - Update every 90 days
   - Revoke old tokens immediately

3. **Use least privilege**
   - Only grant necessary permissions
   - Use organization tokens when possible

4. **Store tokens securely**
   - Use password managers
   - Use environment variables
   - Never share in chat/email

## üìã **Token Usage**

### **Token 1: GitHub Personal Access Token**
- **Purpose:** Repository creation and management
- **Permissions needed:**
  - `repo` (full control)
  - `admin:org` (if creating org repos)

### **Token 2: Cursor/Docker Token**
- **Purpose:** Development tools
- **Keep separate from repository token**
