## 5) Operator workflow (daily)

1. Open current day dashboard.
2. Update standup snapshot and lane statuses.
3. Update ticket board with status/% and evidence links.
4. Update blocker log and dependency tracker.
5. Check gate progression and risks at end of day.
6. Append dashboard change-log line.

## 6) Gate checklist (Sprint 1)

- Gate A: `LI-001` + `LI-002` complete and verified.
- Gate B: `LI-004` API routes pass contract tests.
- Gate C: `LI-005` UI flows pass smoke + lint/tests.
- Gate D: `LI-007` planning transitions tested + audited.
- Gate E: Sprint closure note + carry-over prepared.

## 8) End-of-sprint closeout steps

1. Confirm gate statuses A-E.
2. Ensure all completed tickets have evidence references.
3. Split unfinished tickets into explicit carry-over tasks.
4. Record closure summary in tracker update log.
5. Link final day dashboard and closure note in tracker.
