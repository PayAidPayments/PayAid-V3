## How to read "pending" items

- **Phase-wise rows (`[ ]` / `[x]`)** describe whole blueprint **modules** (`Done` / `Partial` / `Missing`). Most boxes stay `[ ]` until that module is closed at blueprint depth, not because nothing has shipped.
- **P0–P4 backlog lists** are the **ordered roadmap**. Open boxes mean the work package is not finished to blueprint standard, even when parts of the capability already exist in code.
- **Update log (append only)** is the **evidence trail**: commands run and artifact paths. For "what happened lately", scan the newest dated lines here instead of counting open checkboxes.
- **Test gate** applies **per change** you ship; it is not a standing daily checklist for this file alone.
